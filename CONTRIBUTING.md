# 贡献指南

感谢你考虑为本项目贡献代码、文档或想法。

## 开始前

- 先阅读 `README.md`
- 先确认你的改动是否会影响本地数据库路径、导出格式或测试用例
- 如果只是修复或优化一个小点，建议保持改动范围尽量小

## 开发环境

1. 安装 `uv`
2. 执行 `uv sync`
3. 执行 `uv run pytest` 确认基础环境可用

## 常用检查

- `uvx ruff format .`
- `uvx ruff check .`
- `uv run pytest`

## 提交规范

- 使用 Conventional Commits
- 允许的类型：`feat`、`fix`、`docs`、`style`、`refactor`、`perf`、`test`、`chore`
- 提交主题可以使用中文或英文，建议简洁明确
- 推荐格式示例：`feat: 支持自定义数据库路径`

## Pull Request 建议

- 描述清楚改动动机
- 如果是修复问题，说明复现方式和验证结果
- 如果是新增功能，最好附带示例或测试

## 文档修改

- 中文用户优先，文档可以保持中文主导
- 如果你要补充英文说明，建议只放在 README 的摘要或关键入口位置，避免维护负担过重

## 反馈问题

欢迎直接提 Issue。若问题和你的本地环境有关，请尽量附上：

- 操作系统
- Python 版本
- `uv` 版本
- 数据库路径形式
- 复现步骤
