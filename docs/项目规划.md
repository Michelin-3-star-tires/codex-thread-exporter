# 项目规划

## 需求分析

用户希望把 `.codex` 里的线程对话记录查出来，并导出成更适合阅读和归档的 Markdown 文档，方便后续复盘和保存。

具体需求包括：

- 根据线程前缀进行检索
- 在多个匹配结果中手工选择一个或全部
- 导出对话正文，尽量减少过程噪音
- 保留必要的时间戳，便于回溯

## 技术选型理由

- 使用 Python：适合直接读取 SQLite、解析 JSONL 和生成 Markdown
- 使用标准库优先：减少依赖，便于长期维护
- 使用 `uv`：符合当前工作区的依赖管理习惯
- 使用交互式 CLI：比临时脚本更适合反复查询
- 使用通用数据库定位策略：优先支持 `--db-path` 和 `CODEX_STATE_DB`，避免把个人机器路径写进代码

## 架构设计

- `thread_exporter/db.py`：负责读取 SQLite 和解析 rollout JSONL
- `thread_exporter/exporter.py`：负责 Markdown 组织与落盘
- `thread_exporter/cli.py`：负责交互式输入和导出流程
- `pyproject.toml`：负责项目元数据和命令行入口暴露
- `.github/workflows/ci.yml`：负责基础持续集成验证
- `.github/ISSUE_TEMPLATE/` 和 `.github/pull_request_template.md`：负责 Issue 和 PR 协作入口
- `CONTRIBUTING.md`：负责贡献流程、提交规范和协作约定
- `tests/`：负责验证检索与导出格式
