# 项目规划

## 需求分析

用户希望把 `.codex` 里的线程对话记录查出来，并导出成带时间戳的 Markdown 文档，方便后续归档和阅读。

具体需求包括：

- 根据线程前缀进行检索
- 在多个匹配结果中手工选择一个或全部
- 导出完整对话
- 保留时间戳和关键线程参数

## 技术选型理由

- 使用 Python：适合直接读取 SQLite、解析 JSONL 和生成 Markdown
- 使用标准库优先：减少依赖，便于长期维护
- 使用 `uv`：符合当前工作区的依赖管理习惯
- 使用交互式 CLI：比临时脚本更适合反复查询

## 架构设计

- `thread_exporter/db.py`：负责读取 SQLite 和解析 rollout JSONL
- `thread_exporter/exporter.py`：负责 Markdown 组织与落盘
- `thread_exporter/cli.py`：负责交互式输入和导出流程
- `tests/`：负责验证检索与导出格式
